# --fail 功能完整说明

## 概述

新增了 `--fail` 功能，用于处理打包过程中的所有失败情况。当打包失败时，系统会自动调用 `--fail` 回调函数并立即终止打包流程，同时设置正确的退出码。

## 主要改进

### 1. --fail 回调功能
- **自动触发**: 当打包过程中任何步骤失败时，自动调用 `--fail` 功能
- **立即终止**: 调用 `--fail` 后立即终止打包流程，不会继续执行后续步骤
- **详细错误信息**: 提供详细的错误信息，包括错误类型、产品名称、时间戳等

### 2. 正确的退出码 ✨（重要）
- **成功**: 退出码 0
- **失败**: 退出码 1  
- **用户中断**: 退出码 130

这解决了你在 CI/CD 脚本中无法检测到打包失败的问题！

### 3. 兼容性处理
- 自动检测 `PackFile.py` 是否支持 `--fail` 参数
- 如果不支持，使用备用方式显示错误信息
- 仍然正确设置退出码，确保 CI/CD 脚本能检测到失败

## 错误信息格式

`--fail` 接收的错误信息为 JSON 格式：

```json
{
    "error": "具体错误信息",
    "error_type": "错误类型",
    "product": "产品名称",
    "desc": "打包描述",
    "timestamp": "错误发生时间戳"
}
```

## 触发场景

`--fail` 功能会在以下情况下被触发：

1. **配置文件问题**: 无法读取 `config.py` 或 `build-profile.json5`
2. **产品配置问题**: 无法获取或选择产品配置
3. **清理操作失败**: `hvigorw clean` 命令执行失败
4. **同步操作失败**: `hvigorw --sync` 命令执行失败
5. **构建失败**: `hvigorw assembleHap assembleHsp` 命令执行失败
6. **签名失败**: Hap/Hsp 文件签名过程失败
7. **应用信息读取失败**: 无法读取 `AppScope/app.json5` 文件
8. **版本信息获取失败**: 无法获取必要的版本信息
9. **Manifest处理失败**: 创建或签名 `manifest.json5` 文件失败
10. **模板处理失败**: HTML 模板处理过程失败

## Shell 脚本使用示例

现在你的脚本可以正确检测到打包失败：

```bash
function buildAPP() {
    local useHpack=$1
    ohpm clean
    ohpm install
    echo "使用hpack构建..."
    
    # 现在可以正确检测到打包失败！
    if ! hpack pack "test"; then
        echo "❌ hpack 打包失败，终止 CI"
        exit 1
    fi
    
    echo "✅ hpack 打包成功"
}
```

## 自定义失败处理

你可以在 `PackFile.py` 文件中的 `failPack` 函数中自定义失败处理逻辑：

```python
def failPack(errorInfo):
    """打包失败回调，处理打包失败后的错误信息"""
    print("============打包失败信息:============")
    print(json.dumps(errorInfo, indent=4, ensure_ascii=False))
    print("================================")
    print("打包失败，请检查错误信息并修复问题后重试")
    
    # 在这里添加你的自定义处理逻辑
    # 例如：
    # - 发送钉钉/企微通知
    # - 记录错误日志到文件
    # - 上传错误信息到监控系统
    # - 发送邮件通知
```

## 完整功能对比

| 功能 | 调用时机 | 退出码 | 说明 |
|------|----------|--------|------|
| `--will` | 打包前 | - | 可以传递参数给打包脚本 |
| `--did` | 打包成功后 | 0 | 通常用于上传结果、发送通知 |
| `--fail` | 打包失败时 | 1 | 处理错误信息、发送失败通知 ✨ |
